Learning Objectives

  • Connect a static website to a working payment flow
  • Use LLM prompting to generate and revise application code
  • Integrate a payment processor appropriately for a small web application
  • Demonstrate that working software must be testable, reviewable, and verifiable
  • Make a design choice between a hosted checkout model and an API-based approach

Required Deliverables

  • Live Website URL — publicly accessible, no broken links
  • Visible payment button or link on the homepage or main navigation
  • Working credit card payment flow — amount clearly shown as $1.00
  • Success/confirmation page showing item, amount, and date/time or transaction message

Rubric & Scoring
